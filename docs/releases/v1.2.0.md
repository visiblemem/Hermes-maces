## Shadow-first installation

```bash
hermes profile use default
hermes plugins install jefferyzkj01/Hermes-maces --no-enable
```

Back up the active profile config, copy the reviewed settings from `config.example.yaml`, keep `shadow_mode: true`, and allowlist only two or three safe retrieval query fields. Then:

```bash
hermes plugins enable hermes-maces --no-allow-tool-override
hermes gateway restart
```

Run at least seven days of Shadow and complete the validation report before considering bounded influence. Every profile requires separate validation.

## Safety boundaries

- MACES is advisory state, never Canon or a memory provider.
- Hindsight, Obsidian, Session SQLite, `USER.md`, and `MEMORY.md` remain independent and must not be modified.
- The plugin registers no model tools and never originates tool calls.
- Profile identity derives only from trusted `ctx.profile_name` and the active Hermes profile home.
- All text-bearing persistence crosses the central privacy scrubber.
- Failed, cancelled, denied, ambiguous, unlisted, or unsafe tool calls do not create positive learning signals.
- Passive Traditional Chinese learning remains candidate-first and cross-session validated.
- Staged content never enters influence; promotion remains proposal-only.

## Data location and cleanup

MACES state exists only at:

```text
<active-profile-HERMES_HOME>/data/maces/
```

The database is `subconscious.db` with SQLite WAL/SHM companions when active. The plugin checkout contains no live database or user config.

To disable:

```bash
hermes plugins disable hermes-maces
hermes gateway restart
```

Confirm Hermes, Hindsight, Obsidian, Session SQLite, and profile memory are normal. Preserve the MACES database for local investigation. To remove:

```bash
hermes plugins remove hermes-maces
```

Only while the gateway is stopped and the profile path is confirmed, delete `<active-profile-HERMES_HOME>/data/maces/`. Never delete Hindsight, Obsidian, Session SQLite, `USER.md`, `MEMORY.md`, or Hermes core files as part of MACES rollback.

## Known limitations

- Patterns are speculative advisory signals, not facts.
- Strict learning gates may produce little data; this is expected.
- Traditional Chinese extraction uses bounded rules and is not full semantic understanding.
- Each profile must complete a separate Shadow evaluation.
- Unsafe user-added allowlist fields can increase privacy risk.
- `main` is the actual shallow-clone installer source; read the Changelog before updates.
- No cloud sync, cross-device synchronization, or remote backup is provided.
- Real multi-user and longer-duration validation is still required before a stable label.
